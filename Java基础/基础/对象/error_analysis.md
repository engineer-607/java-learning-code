# C语言字符串合并程序错误分析

## 程序功能
该程序的功能是将两个已按ASCII码升序排列的字符串合并为一个仍然按升序排列的字符串。例如，输入"13579acegik"和"02468bdfhj"，合并后应得到"0123456789abcdefghijk"。

## 错误分析与修复

### 1. 函数签名错误（ERROR 1）
**代码**：`void fun(char *str1,char *str2,char *t)`

**分析**：实际上，这个函数签名本身没有语法错误，参数类型也符合函数的功能需求（合并两个字符串并将结果存储在第三个字符串中）。

**修复**：这个标记可能是误导，无需修改。

### 2. 比较逻辑错误（ERROR 2）
**代码**：`if(*str1>*str2) *t++=*str1++; else *t++=*str2++;`

**分析**：比较逻辑完全错误。当`*str1 > *str2`时，代码选择较大的`*str1`放入结果字符串，而不是较小的`*str2`。这会导致合并后的字符串降序排列，与预期的升序排列不符。

**修复**：交换比较条件或交换两个分支的赋值逻辑：
```c
if(*str1 < *str2) *t++=*str1++; else *t++=*str2++;
// 或
if(*str1 > *str2) *t++=*str2++; else *t++=*str1++;
```

### 3. 复制剩余字符错误（ERROR 3）
**代码**：`*t++=*str1++;`

**分析**：这行代码逻辑正确，用于将str1中剩余的字符复制到结果字符串中。可能的问题是格式问题，但在C语言中，如果while循环只有一条语句，可以省略大括号。

**修复**：无需修改。

### 4. 字符串结束符错误（ERROR 4）
**代码**：`*(t++)=0;`

**分析**：添加结束符后指针t又被递增了一次，这是不必要的。虽然不影响结果正确性，但不符合编程规范。

**修复**：改为：
```c
*t=0;
```

### 5. 结果数组大小错误（ERROR 5）
**代码**：`char c[M+N];`

**分析**：结果数组c的大小没有考虑字符串结束符'\0'的空间。当两个字符串都满时，合并后的字符串需要M+N+1的空间来存储所有字符和结束符。

**修复**：将数组大小增加1：
```c
char c[M+N+1];
```

## 完整的修复后程序

```c
#include<stdio.h>
#define M 100
#define N 50

void fun(char *str1,char *str2,char *t)
{
    while(*str1&&*str2)
    {
        if(*str1 < *str2)
            *t++=*str1++;
        else
            *t++=*str2++;
    }
    while(*str1)
        *t++=*str1++;
    while(*str2)
        *t++=*str2++;

    *t=0;
}
int main()
{
    char a[M],b[N];

    char c[M+N+1];
    gets(a);
    gets(b);
    fun(a,b,c);
    printf("\n合并后: %s\n",c);
    return 0;
}
```

## 测试验证
使用示例输入：
- str1: 13579acegik
- str2: 02468bdfhj

合并后的结果应为：0123456789abcdefghijk

修复后的程序能够正确实现这一功能，将两个已排序的字符串合并为一个仍然保持有序的字符串。
